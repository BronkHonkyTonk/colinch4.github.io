### 서비스의 보안 설정이 중요한 이유

일반적으로 서버에는 다양한 서비스가 실행되며, 이러한 서비스들은 해킹 시도의 대상이 될 수 있습니다. 따라서, 서비스의 적절한 보안 설정은 매우 중요합니다. 만약 취약한 서비스 설정으로 공격자에게 노출된다면, 시스템에 심각한 피해가 발생할 수 있습니다.

### 서비스 보안을 강화하는 방법

서비스 보안을 강화하는 방법에는 다음과 같은 것들이 있습니다:

- **파이어월 구성 및 필터링**: 서비스가 열려있는 포트를 제한하고, 불필요한 트래픽을 차단하여 서비스에 접근하는 것을 방지합니다.
- **권한 관리**: 서비스가 실행되는 사용자의 권한을 최소화하여 공격 시 피해를 최소화할 수 있습니다.
- **암호화 사용**: 서비스 간에 데이터를 전송할 때 암호화를 적용하여 중간자 공격을 방지할 수 있습니다.

### 서비스의 보안 설정 예시: SSH

SSH(보안 셸)는 리눅스 시스템에서 사용자 인증 및 보안적인 원격 접속을 가능하게 하는 프로토콜입니다. SSH의 보안 설정을 강화하기 위해 다음과 같은 작업을 수행할 수 있습니다:

1. **기본 포트 변경**: SSH 접속을 위한 기본 포트를 변경하여 봇에 의한 자동 접속 시도를 방지할 수 있습니다.
   ```bash
   # /etc/ssh/sshd_config 파일 내에서 포트 번호 변경
   Port 2222
   ```

2. **비밀번호 인증 비활성화**: 키 기반(공개키 및 개인키)의 인증 방식을 사용하여 패스워드 인증을 비활성화할 수 있습니다.
   ```bash
   # /etc/ssh/sshd_config 파일 내에서 비밀번호 인증 비활성화
   PasswordAuthentication no
   ```

3. **루트 사용자의 SSH 접속 제한**: 루트 사용자의 SSH 접속을 제한하여 보안을 강화할 수 있습니다.
   ```bash
   # /etc/ssh/sshd_config 파일 내에서 루트 사용자의 SSH 접속 제한
   PermitRootLogin no
   ```

---

위의 과정을 통해, 서비스의 보안 설정을 강화할 수 있습니다. 이러한 설정은 시스템의 안정성을 유지하고 보안을 강화하는 데 도움이 됩니다.
