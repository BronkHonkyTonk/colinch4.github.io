리눅스나 유닉스 시스템에서는 주기적으로 실행되어야 하는 작업을 자동으로 처리하기 위해 **cron**을 사용할 수 있습니다. **cron**은 시간 기반으로 작업을 예약해서 실행하는 스케줄링 시스템입니다. 이를 이용하여 쉘 스크립트를 주기적으로 실행하는 방법을 살펴봅시다.

## 쉘 스크립트 작성

먼저 주기적으로 실행할 쉘 스크립트를 작성합니다. 예를 들어, `backup.sh`라는 스크립트를 작성한다고 가정해봅시다. 이 스크립트는 백업을 수행하는 역할을 합니다.

```bash
#!/bin/bash
# backup.sh

# 여기에 백업 작업을 수행하는 명령어를 작성합니다.
```

## cron 작업 등록

다음으로, 이 스크립트를 주기적으로 실행할 cron 작업을 등록해야 합니다. `crontab -e` 명령어를 사용하여 cron 작업을 편집할 수 있습니다.

```bash
$ crontab -e
```

이 명령어를 실행하면 기본 텍스트 편집기가 열리며, 아래와 같이 주기적으로 스크립트를 실행하는 작업을 추가할 수 있습니다.

```
# 매일 오전 3시에 backup.sh 스크립트 실행
0 3 * * * /bin/bash /path/to/backup.sh
```

위 예시에서 `0 3 * * *`는 매일 오전 3시를 나타내며, `/bin/bash /path/to/backup.sh`는 실행할 스크립트 경로를 나타냅니다. 이렇게 작성한 작업은 매일 오전 3시에 `backup.sh`를 실행합니다.

## cron 작업 확인

cron 작업이 제대로 등록되었는지 확인하기 위해서는 `crontab -l` 명령어를 사용할 수 있습니다.

```bash
$ crontab -l
```

이 명령어를 실행하면 현재 유저의 cron 작업 목록을 확인할 수 있습니다.

`crontab`을 이용하여 주기적인 쉘 작업을 예약하는 방법을 알아보았습니다. 이를 통해 일일 작업이나 백업 등을 자동으로 처리할 수 있습니다.
