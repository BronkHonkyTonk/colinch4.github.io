리눅스 시스템에서 네트워크 패킷이 전달되는 방법을 결정하는 데 사용되는 라우팅 테이블은 중요한 요소입니다. 이 게시물에서는 리눅스 라우팅 테이블을 설정하는 방법을 살펴보겠습니다.

## 1. 현재 라우팅 테이블 확인

먼저 현재 시스템의 라우팅 테이블을 확인해야 합니다. 이를 위해 다음 명령어를 사용합니다:

```bash
$ ip route show
```

위 명령어를 실행하면 현재의 라우팅 테이블이 나열됩니다.

## 2. 라우팅 테이블에 항목 추가

라우팅 테이블에 새로운 항목을 추가하려면 `ip route add` 명령어를 사용합니다. 예를 들어, 특정 대상 네트워크로의 패킷을 다른 게이트웨이로 라우팅하고 싶다면 다음과 같이 명령어를 사용할 수 있습니다:

```bash
$ sudo ip route add 192.168.1.0/24 via 10.0.0.1
```

이렇게 하면 대상 네트워크가 192.168.1.0/24인 패킷이 10.0.0.1 게이트웨이를 통해 전달됩니다.

## 3. 라우팅 테이블에서 항목 삭제

필요 없는 라우팅 항목을 제거하려면 `ip route del` 명령어를 사용합니다. 예를 들어, 이전에 추가한 항목을 삭제하려면 다음과 같이 명령어를 사용할 수 있습니다:

```bash
$ sudo ip route del 192.168.1.0/24 via 10.0.0.1
```

이렇게 하면 해당 라우팅 항목이 삭제됩니다.

## 요약

라우팅 테이블을 올바르게 설정하고 유지하는 것은 네트워크 관리의 중요한 부분입니다. 새로운 라우팅 항목을 추가하거나 삭제함으로써 리눅스 시스템의 라우팅 동작을 조정할 수 있습니다.

이로써, 리눅스 라우팅 테이블 설정에 대한 간단한 안내를 마칩니다.
