리눅스 서버에서는 **네트워크 백업**이 중요한 작업 중 하나이다. 네트워크를 통해 데이터를 안전하게 백업하고 복원하는 방법을 알아보자.

## 1. 백업 설정

### 1.1. rsync를 사용한 네트워크 백업

리눅스에서는 `rsync`를 사용하여 네트워크를 통해 다른 서버로 파일을 동기화하는 방법이 있다. 예를 들어, 다음과 같이 실행할 수 있다.

```bash
rsync -avz /원본/경로/ 계정@호스트:/백업/경로/
```

이 명령은 `/원본/경로/`의 파일을 `계정@호스트`의 `/백업/경로/`로 동기화한다.

### 1.2. SCP를 사용한 백업

또 다른 방법은 `SCP` (Secure Copy Protocol)을 사용하여 파일을 안전하게 복사할 수 있다.

```bash
scp /원본/경로/ 계정@호스트:/백업/경로/
```

## 2. 복원 설정

### 2.1. rsync를 사용한 네트워크 복원

백업된 파일을 복원할 때, `rsync`를 이용하여 네트워크를 통해 복원할 수 있다.

```bash
rsync -avz 계정@호스트:/백업/경로/ /복원/경로/
```

### 2.2. SCP를 사용한 복원

백업된 파일을 `SCP`를 사용하여 다시 로컬로 가져올 수 있다.

```bash
scp 계정@호스트:/백업/경로/ /복원/경로/
```

---

이제, **리눅스 서버**에서 네트워크를 통해 안전하게 백업하고 복원하는 설정을 알게 되었다. 데이터의 안전은 운영체제에서 가장 중요한 부분이므로, 꼭 검토하고 활용하기 바란다.
