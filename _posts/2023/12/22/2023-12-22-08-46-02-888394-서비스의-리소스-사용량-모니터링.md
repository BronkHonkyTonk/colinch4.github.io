리눅스 서버에서 실행 중인 서비스의 리소스 사용량을 모니터링하는 것은 시스템 성능을 파악하고 문제를 신속하게 해결하는 데 매우 중요합니다. 이를 통해 리소스에 대한 실시간 정보를 확인하고 성능 이슈를 대비할 수 있습니다.

## 1. **top 명령어를 사용한 리소스 모니터링**

[top](https://linux.die.net/man/1/top) 명령어는 실행 중인 프로세스의 실시간 리소스 사용량을 보여주는 명령어입니다. 간단히 `top`을 실행하면 CPU 및 메모리 사용량, 프로세스 목록 등을 실시간으로 확인할 수 있습니다.

```bash
top
```

이 외에도 `top` 명령어는 다양한 옵션을 제공하므로 해당 옵션을 사용하여 사용자 정의 모니터링을 구성할 수 있습니다.

## 2. **htop을 사용한 대화형 리소스 모니터링**

[htop](https://hisham.hm/htop/)은 top 명령어의 확장판으로, 대화형으로 실행 중인 프로세스의 리소스 사용량을 시각적으로 파악할 수 있는 툴입니다. `htop`은 보다 직관적이고 사용자 친화적인 인터페이스를 제공하여 효율적인 모니터링이 가능합니다.

```bash
htop
```

`htop`은 CPU, 메모리, 스토리지 사용량 뿐만 아니라, 프로세스 관리 및 정렬 기능 등을 제공하여 리소스 관리를 용이하게 합니다.

## 3. **시스템 리소스 사용량 그래픽으로 확인하기**

[GNOME System Monitor](https://help.gnome.org/users/gnome-system-monitor/stable/)나 [KSysGuard](https://kde.org/applications/system/ksysguard/)와 같은 그래픽 도구를 사용하여 시스템 리소스 사용량을 시각적으로 모니터링할 수 있습니다.

이러한 도구들은 CPU, 메모리, 네트워크, 디스크 사용량을 그래픽으로 제공하며, 시스템 상태를 실시간으로 확인하고 모니터링할 수 있습니다.

## 결론

서비스의 리소스 사용량을 모니터링함으로써 시스템의 성능을 정확히 파악하고, 문제가 발생할 때 빠르게 대응할 수 있습니다. `top`, `htop` 및 그래픽 도구들을 적절히 활용하여 서버의 리소스 사용량을 실시간으로 확인하고, 최적의 성능을 유지할 수 있습니다.
