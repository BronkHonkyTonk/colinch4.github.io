자바에서 상속은 객체지향 프로그래밍의 핵심 개념 중 하나입니다. 상속을 사용하면 기존 클래스의 속성과 메서드를 다른 클래스에서 재사용할 수 있으며, 새로운 클래스를 만들 때 중복을 피하고 코드를 간결하게 유지할 수 있습니다. 아래는 자바에서 상속을 활용한 다양한 사례들을 살펴보겠습니다.

### 1. 확장 클래스

상속을 사용하여 새로운 클래스를 기존 클래스의 확장으로 정의할 수 있습니다. 예를 들어, `Vehicle` 클래스가 있고 이를 확장한 `Car`와 `Bike` 클래스를 만들 수 있습니다. 이렇게 하면 `Car`와 `Bike` 클래스는 `Vehicle` 클래스의 속성과 메서드를 상속받아 사용할 수 있습니다.

```java
public class Vehicle {
    // 속성과 메서드들
}

public class Car extends Vehicle {
    // Vehicle 클래스의 속성과 메서드를 상속받음
}

public class Bike extends Vehicle {
    // Vehicle 클래스의 속성과 메서드를 상속받음
}
```

### 2. 메서드 오버라이딩

하위 클래스에서 상위 클래스의 메서드를 재정의하여 사용할 수 있습니다. 예를 들어, `Vehicle` 클래스에 `start()` 메서드가 있고, `Car` 클래스에서는 이를 재정의하여 자동차를 시동 거는 행동에 맞게 구현할 수 있습니다.

```java
public class Vehicle {
    public void start() {
        // 일반적인 시작 행동
    }
}

public class Car extends Vehicle {
    @Override
    public void start() {
        // 자동차의 경우의 시작 행동으로 재정의
    }
}
```

### 3. 다형성 활용

상속을 통해 여러 클래스를 하나의 공통된 타입으로 관리할 수 있습니다. 이를 통해 다형성을 활용하여 유연하고 간결한 코드를 작성할 수 있습니다. 

```java
Vehicle car = new Car();
Vehicle bike = new Bike();
```

### 결론

상속은 자바에서 객체지향 프로그래밍의 강력한 도구 중 하나이며, 코드의 재사용성과 유지보수성을 높일 수 있는 다양한 기능을 제공합니다. 적절하게 활용하면 객체지향 프로그래밍의 장점을 최대로 발휘할 수 있습니다.

이처럼 상속은 다양한 사용 사례를 가지고 있으며, 제대로 활용하기 위해서는 상속의 개념과 규칙을 잘 이해해야 합니다.

많은 상황에서 상속을 대체할 수 있는 다른 방법들도 있으므로, 해당 상황에 가장 적합한 설계 방식을 선택하는 것이 중요합니다.
