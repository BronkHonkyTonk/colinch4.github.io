1. **고유성 규칙**: 각 레코드가 유일한 값을 가지도록 제약을 설정하여 중복 데이터를 방지합니다.
2. **외래 키 규칙**: 다른 테이블의 레코드를 참조하는 외래 키 관계를 설정하여 데이터 일관성을 유지합니다.
3. **체크 규칙**: 데이터의 유효성을 검증하기 위해 조건을 설정하여 잘못된 데이터의 삽입 또는 수정을 방지합니다.
4. **기본 키 규칙**: 각 레코드를 고유하게 식별하기 위한 기본 키 제약을 설정하여 데이터 무결성을 보장합니다.
5. **트리거 규칙**: 특정 이벤트가 발생할 때 자동으로 실행되는 작업을 정의하여 데이터의 일관성을 유지합니다.

이러한 표준 규칙을 준수하여 SQL 데이터베이스를 관리하면 데이터의 무결성과 안정성을 보다 효과적으로 유지할 수 있습니다.
