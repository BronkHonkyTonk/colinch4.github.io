행위 다이어그램은 주로 세 가지 종류로 나뉩니다: 사용 사례 다이어그램, 시퀀스 다이어그램, 상태 다이어그램입니다. 이 중에서 상태 다이어그램은 객체의 생명주기를 모델링하고, 시퀀스 다이어그램은 객체 간의 상호 작용을 보여주며, 사용 사례 다이어그램은 시스템의 기능적 요구사항을 보여줍니다.

아래는 C++에서의 간단한 행위 다이어그램의 예제 코드입니다.

```cpp
class Object {
public:
    void action1() {}
    void action2() {}
};

int main() {
    Object obj;
    
    obj.action1();
    obj.action2();
    
    return 0;
}
```

행위 다이어그램은 업무 또는 소프트웨어 시스템의 동작을 이해하고 시각적으로 표현하기 위한 유용한 도구입니다. UML 표준의 일부분으로 많은 소프트웨어 개발 환경에서 지원되며, 팀 간의 의사 소통과 시스템 설계에 도움이 됩니다.
