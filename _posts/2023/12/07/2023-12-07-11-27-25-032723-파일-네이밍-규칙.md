1. 소문자 사용: JavaScript 파일은 소문자로 작성해야 합니다. 대문자는 특정 운영 체제에서 문제를 발생시킬 수 있습니다. 예를 들어 "MyFile.js" 대신 "myfile.js"와 같이 작성해야 합니다.

2. 하이픈(-) 사용: 파일 이름에 공백이 필요한 경우 하이픈을 사용하세요. 예를 들어 "main.js" 대신 "main-script.js"와 같이 작성합니다. 공백 대신 언더스코어(_)나 카멜 케이스(CamelCase)를 사용하는 것도 가능하지만, 하이픈을 사용하는 것이 가독성면에서 더 좋습니다.

3. 명확하고 서술적인 이름: 파일 이름은 해당 파일의 역할을 명확하게 표현해야 합니다. 다른 개발자나 향후 자신이 해당 파일을 보았을 때, 내용을 파악하기 쉬워야합니다. 예를 들어, "utils.js"와 같이 모호한 이름보다는 "date-utils.js"나 "validation-helpers.js"와 같이 더 구체적인 이름을 사용하는 것이 좋습니다.

4. 파일 확장자: JavaScript 파일은 ".js" 확장자를 사용합니다. 이는 파일이 JavaScript 코드임을 알리는 데 도움을 주고 다른 개발자들에게 코드 타입을 알려줍니다.

아래에 몇 가지 예를 보여드리겠습니다.

- main.js
- utils.js
- date-utils.js
- validation-helpers.js
- app-constants.js

JavaScript 파일 네이밍 규칙을 따르면 프로젝트의 코드 구성과 유지 보수가 쉬워집니다. 이러한 규칙을 적용하여 효과적인 파일 네이밍을 실천해보세요!
