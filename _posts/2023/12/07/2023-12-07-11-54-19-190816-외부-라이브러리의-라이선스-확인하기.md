## 소개
안드로이드 앱 또는 Kotlin 프로젝트를 개발할 때, 종종 다양한 외부 라이브러리를 사용하게 됩니다. 하지만 이러한 라이브러리들은 각각 다른 라이선스를 가지고 있을 수 있으며, 이를 확인해야 합니다. 왜냐하면 일부 라이브러리의 라이선스가 프로젝트의 라이선스와 충돌할 수도 있기 때문입니다. 

이번 포스트에서는 Kotlin 프로젝트에서 외부 라이브러리의 라이선스를 확인하는 방법을 알아보겠습니다.

## 방법

### 1. 라이브러리의 README 파일 확인하기
가장 간단하고 빠른 방법은 라이브러리의 README 파일을 확인하는 것입니다. 보통 README 파일에는 라이브러리의 사용 방법 또는 문서와 함께 라이선스 정보가 포함되어 있습니다. README 파일을 통해 라이선스를 확인할 수 있습니다.

### 2. 라이브러리의 소스 코드 확인하기
만약 README 파일에서 라이선스 정보를 찾을 수 없는 경우, 라이브러리의 소스 코드를 직접 확인하는 방법을 사용할 수 있습니다. 일반적으로 외부 라이브러리는 오픈 소스로 개발되었기 때문에 라이선스 정보가 소스 코드 내에 주석으로 포함되어 있을 수 있습니다. 소스 코드를 열어서 라이선스 정보를 찾아보세요.

### 3. 라이브러리의 매니페스트 파일 확인하기
때로는 라이브러리의 매니페스트 파일에 라이선스 정보가 포함되어 있는 경우도 있습니다. 프로젝트에서 사용하는 모든 라이브러리의 매니페스트 파일을 확인해보세요.

### 4. 온라인 리소스 이용하기
마지막으로, 온라인 리소스를 활용하여 라이브러리의 라이선스 정보를 확인할 수 있습니다. 종종 라이브러리의 공식 웹사이트 또는 오픈 소스 프로젝트 사이트에서 라이선스 정보를 확인할 수 있습니다. 또한 라이선스 관련 정보를 조회할 수 있는 온라인 도구나 데이터베이스도 있습니다.

## 결론
Kotlin 프로젝트에서 외부 라이브러리의 라이선스를 확인하는 것은 중요한 작업입니다. 이를 통해 프로젝트의 라이선스와 라이브러리의 라이선스와의 충돌을 방지할 수 있습니다. 위에서 소개한 방법들을 활용하여 외부 라이브러리의 라이선스를 쉽게 확인할 수 있습니다.
