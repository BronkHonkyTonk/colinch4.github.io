SQL 데이터베이스는 로그인한 후에도 특정 시간 동안 사용자를 로그인 상태로 유지할 수 있습니다. 하지만, 보안 및 자원 관리를 위해 로그아웃 시간 제한을 설정하는 것이 중요합니다. 이 글에서는 SQL 데이터베이스의 로그아웃 시간 제한을 설정하는 방법을 알아보겠습니다.

## 1. 데이터베이스 관리 시스템 접속

먼저 SQL 데이터베이스를 관리하는 데이터베이스 관리 시스템에 접속해야 합니다. 이 글에서는 MySQL 데이터베이스를 예시로 설명하겠습니다.

## 2. 로그아웃 시간 제한 설정

MySQL 데이터베이스에서 로그아웃 시간 제한을 설정하려면 "wait_timeout" 변수를 수정해야 합니다. 이 변수는 클라이언트와의 접속이 끊어지기 전에 대기 시간을 초 단위로 지정합니다.

아래의 방법을 따라 진행하면 됩니다:
1. SQL 쿼리 창을 엽니다.
2. 다음 쿼리를 실행하여 "wait_timeout" 변수 값을 원하는 값(초 단위)으로 수정합니다.
   ```sql
   SET GLOBAL wait_timeout = 3600;
   ```
   위의 쿼리는 "wait_timeout" 값을 1시간(3600초)로 설정하는 예시입니다.

## 3. 변경 내용 저장 및 적용

마지막으로, 변경된 설정을 저장하고 적용해야 합니다. 이를 위해 데이터베이스 관리 시스템의 설정을 저장하고 재시작하는 방법을 사용하면 됩니다. 재시작 없이도 변경사항을 즉시 적용할 수 있는 경우도 있습니다.
