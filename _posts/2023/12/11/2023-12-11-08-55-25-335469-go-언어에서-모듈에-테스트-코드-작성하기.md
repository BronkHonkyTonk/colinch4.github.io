Go 언어에서 모듈을 개발할 때, 테스트 코드를 작성하여 모듈의 동작을 확인하는 것은 매우 중요합니다. 이번 글에서는 Go 언어로 모듈을 개발하면서 테스트 코드를 작성하는 방법에 대해 알아보겠습니다.

## 테스트 코드 작성하기

Go 언어에서 테스트 코드를 작성하기 위해서는 `testing` 패키지를 활용합니다. 일반적으로 테스트 파일의 이름은 `_test.go`로 끝나며, 해당 파일에는 `Test`로 시작하는 함수들을 작성합니다. 

아래는 간단한 예제 코드입니다.

```go
package example

import "testing"

func Add(a, b int) int {
    return a + b
}

func TestAdd(t *testing.T) {
    result := Add(2, 3)
    if result != 5 {
        t.Errorf("Add(2, 3)의 결과: %d, 원하는 값: 5", result)
    }
}
```

위 예제 코드에서 `Add` 함수와 `TestAdd` 함수를 작성했습니다. `TestAdd` 함수에서는 `Add` 함수의 결과가 예상한 대로 나오는지 테스트하고 있습니다.

## 테스트 실행하기

테스트 코드를 작성한 후에는 터미널에서 해당 패키지 디렉토리에서 `go test` 명령어를 실행하여 테스트를 실행할 수 있습니다.

```sh
go test
```

테스트가 성공적으로 완료되면, 아무런 출력이 나타나지 않습니다. 하지만, 테스트가 실패한 경우에는 에러 메시지와 함께 실패한 테스트들에 대한 상세한 정보를 확인할 수 있습니다.

## 마무리

Go 언어에서 모듈을 개발할 때, 테스트 코드를 작성하여 모듈의 동작을 확인하는 것은 매우 중요합니다. 위에서 소개한 방법을 활용하여 효과적으로 모듈을 테스트하는 경험을 해보시기 바랍니다.

더 많은 정보를 원하시는 경우에는 Go 언어의 [공식 문서](https://golang.org/pkg/testing/)를 참고하시기 바랍니다.
