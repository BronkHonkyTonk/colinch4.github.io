## 안드로이드 아키텍처 컴포넌트란?

**안드로이드 아키텍처 컴포넌트**는 안드로이드 앱을 더 효율적으로 구축하고 유지보수할 수 있도록 도와주는 라이브러리 집합입니다. 주요 컴포넌트로는 **ViewModel**, **LiveData**, **Room**, **Navigation** 등이 있습니다. 이러한 컴포넌트들은 각자의 역할에 따라 앱의 아키텍처를 보다 간단하고 일관되게 설계할 수 있도록 도와줍니다.

## 안드로이드 앱에서의 사용 사례

### 라이프사이클 관리

안드로이드의 **ViewModel**은 화면 회전과 같은 구성 변경이 발생했을 때 데이터를 보존하고 다시 사용할 수 있도록 해줍니다. 또한, **LiveData**를 사용하면 뷰와 데이터를 더욱 쉽게 동기화할 수 있습니다. 이들은 앱의 라이프사이클에 맞게 데이터를 관리하고 화면 갱신 등의 작업을 처리하는 데 유용합니다.

```kotlin
class MyViewModel : ViewModel() {
    val currentName: MutableLiveData<String> by lazy {
        MutableLiveData<String>()
    }
}
```

### 데이터 관리

**Room 라이브러리**는 SQLite 데이터베이스 위에 구축된 안전한 추상화 레이어를 제공합니다. 이를 통해 데이터베이스와의 작업을 쉽게 처리할 수 있으며, 데이터의 변경 사항에 대한 알림도 LiveData를 통해 받을 수 있습니다.

```kotlin
@Dao
interface UserDao {
    // 사용자 정보 조회
    @Query("SELECT * FROM user")
    fun getAll(): LiveData<List<User>>
    
    // 사용자 정보 추가
    @Insert
    fun insert(user: User)
}
```

### 사용자 인터페이스

**Navigation 컴포넌트**는 화면 간 전환 및 탐색을 관리하는 도구로, 화면 간 이동 및 앱 내 탐색 구조를 보다 효율적으로 관리할 수 있게 해줍니다. 이를 통해 사용자 경험을 향상시키고 개발 생산성을 향상시킬 수 있습니다.

```kotlin
val directions = MyFragmentDirections.actionStartFragment()
Navigation.findNavController(view).navigate(directions)
```
