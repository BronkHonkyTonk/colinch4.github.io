C++에서 네임스페이스는 코드를 구조화하고 이름 충돌을 방지하는 데 사용됩니다. 네임스페이스는 전역 범위에서 식별자의 범위를 제어하고, 코드의 가독성을 향상시키며, 컴파일 시간에 이름 충돌을 방지합니다.

네임스페이스는 기본적으로 확장 가능합니다. 여러 파일과 라이브러리에서 동일한 네임스페이스를 사용하여 기능을 확장할 수 있습니다. 이는 코드를 모듈화하고 재사용성을 높일 수 있는 중요한 장점입니다.

```c++
// Example.h
namespace Example {
    void foo();
}

// Example.cpp
#include "Example.h"
#include <iostream>
void Example::foo() {
    std::cout << "Hello from Example::foo()" << std::endl;
}
```

위의 예시에서는 "Example"라는 네임스페이스를 정의하고, `foo` 함수를 구현합니다. 이를 다른 파일에서 사용하고자 할 때에도 `Example::foo()` 형태로 호출할 수 있습니다.

네임스페이스의 확장성을 활용하면 서로 다른 모듈에서 동일한 네임스페이스를 사용하여 코드를 쉽게 통합할 수 있고, 유지보수하기도 용이해집니다.

네임스페이스를 사용함으로써, 코드의 가독성과 유지보수성을 향상시키고, 이름 충돌을 방지할 수 있습니다.

## 결론
C++의 네임스페이스는 코드의 구조화와 모듈화에 중요한 역할을 합니다. 네임스페이스를 적절히 사용하여 코드를 효율적으로 관리하고 확장 가능한 소프트웨어를 개발할 수 있습니다.
