스프링 부트는 단일 JAR 파일로 애플리케이션을 패키징하여 간단히 배포할 수 있는 장점이 있습니다. 이를 통해 애플리케이션을 독립적으로 실행할 수 있으며, 간편한 배포 및 확장이 가능합니다.

## 1. 스프링 부트 JAR 패키징

스프링 부트 프로젝트를 JAR 파일로 패키징하려면 프로젝트의 루트 디렉토리에서 아래 명령을 실행합니다.

```bash
./mvnw clean package
```

위 명령을 실행하면 Maven Wrapper를 통해 프로젝트를 빌드하고, `target` 폴더에 실행 가능한 JAR 파일이 생성됩니다.

## 2. 외부 설정 파일 포함

스프링 부트 JAR 파일은 실행 시 외부 설정 파일을 포함할 수 있습니다. `application.properties` 또는 `application.yaml`과 같은 외부 설정 파일을 JAR 파일과 동일한 디렉토리에 넣으면, 애플리케이션이 실행될 때 해당 설정 파일이 자동으로 로딩됩니다.

## 3. 배포

JAR 파일을 서버에 배포하고 실행하려면 아래 명령을 사용합니다.

```bash
java -jar <파일명>.jar
```

위 명령을 통해 스프링 부트 애플리케이션이 실행됩니다.

스프링 부트의 JAR 파일 패키징과 해당 JAR 파일을 이용한 배포를 통해 애플리케이션을 쉽고 빠르게 배포할 수 있으며, 이는 생산성과 유연성을 높일 뿐만 아니라 유지보수 및 확장성에도 도움이 됩니다.

이상으로 스프링 부트 프로젝트의 패키징과 배포 방법에 대해 알아보았습니다.

더 많은 정보와 상세한 내용은 [Spring Boot 공식 문서](https://spring.io/projects/spring-boot)를 참고하시기 바랍니다.
