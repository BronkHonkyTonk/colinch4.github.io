소프트웨어 개발에서 코드의 **재사용성**은 매우 중요합니다. 중복 코드를 최소화하고 유지보수를 쉽게 하려면 객체 지향 프로그래밍에서 상속을 활용하는 것이 효율적입니다.

## 상속의 개념
**상속**은 한 클래스가 다른 클래스의 속성과 메서드를 물려받는 것을 의미합니다. 부모 클래스(또는 슈퍼 클래스)의 특성을 자식 클래스(또는 서브 클래스)가 이어받아 재사용할 수 있게 해줍니다.

```swift
class Shape {
    var color: String?
    func draw() {
        // draw shape
    }
}

class Circle: Shape {
    var radius: Double?
    // additional properties and methods for Circle
}
```

위의 예시에서 `Circle` 클래스는 `Shape` 클래스를 상속하여 `color` 속성과 `draw()` 메서드를 사용할 수 있습니다.

## 코드 재사용성 증가
상속을 통해 코드의 재사용성이 높아집니다. 기존의 클래스를 수정하지 않고 새로운 동작을 추가하거나 적응하여 새로운 클래스를 생성할 수 있으며 이는 유지보수 및 확장이 용이하도록 만들어 줍니다.

## 유의사항
상속은 잘못 사용될 경우 코드 구조를 복잡하게 만들 수 있습니다. 또한, 부모 클래스의 변경이 모든 자식 클래스에 영향을 미칠 수 있으므로 주의가 필요합니다.

상속을 사용하는 것이 항상 최선인 것은 아니므로, **컴포지션**과 같은 다른 설계 원칙과 비교하여 상황에 맞게 선택해야 합니다.

상속을 올바르게 사용하면 코드의 재사용성을 향상시키고 유연하고 확장 가능한 소프트웨어를 작성하는 데 도움이 됩니다.
