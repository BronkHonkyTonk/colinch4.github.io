## 소개
JavaScript는 대표적인 웹 프로그래밍 언어 중 하나로 많은 사람들이 사용하고 있습니다. 그러나 프로젝트에서 여러 개발자가 함께 작업하면서 코드의 일관성과 가독성을 유지하기 어려울 수 있습니다. 이런 문제를 해결하기 위해 코드 컨벤션을 통해 표준화된 가이드를 준수하는 것이 중요합니다.

## 가이드라인

### 1. 들여쓰기
- 들여쓰기는 스페이스 2개를 사용합니다.
- 탭 문자를 사용하지 않습니다.

### 2. 변수 및 함수 이름
- 변수 및 함수의 이름은 소문자로 시작하고 카멜 케이스를 사용합니다.
  - 예: `myVariable`, `myFunction`
- 클래스 이름은 대문자로 시작하고 파스칼 케이스를 사용합니다.
  - 예: `MyClass`
  
### 3. 중괄호
- 중괄호는 항상 새로운 줄에 위치합니다.

```javascript
if (condition) {
  // code
}
```

### 4. 줄 바꿈
- 줄의 길이가 80자를 초과할 경우, 줄 바꿈을 해야 합니다.

### 5. 주석
- 주석은 코드의 이해를 돕기 위해 사용되며, 가독성을 향상시킵니다.
- 주석은 `//`를 사용하며, 해당 주석은 단일 라인에만 작성합니다.

```javascript
// 이 부분은 변수를 초기화하는 코드입니다.
let x = 10;
```

### 6. 기타
- 작은 따옴표(`''`)보다는 큰 따옴표(`""`)를 사용하여 문자열을 표기합니다.
- 세미콜론(`;`)은 코드의 끝에 항상 붙입니다.
- 불필요한 공백은 피하고, 코드의 가독성을 높이기 위해 공백을 적절히 사용합니다.

## 결론
이 가이드라인은 JavaScript 코드 컨벤션에 대한 표준화된 가이드를 제시했습니다. 이를 따르면 프로젝트 내에서 일관성 있는 코드를 작성할 수 있으며, 다른 개발자와 협업 시 코드의 가독성을 높일 수 있습니다. 코드 컨벤션 가이드를 준수함으로써 효율적이고 유지보수가 쉬운 JavaScript 코드를 작성할 수 있습니다.
