현대 웹 개발에서 자바스크립트의 역할은 더욱 중요해지고 있습니다. 자바스크립트는 웹 애플리케이션의 동적인 기능을 구현하는 데 사용되며, 서버와의 통신, DOM 조작, 사용자 인터랙션 등 다양한 역할을 수행합니다. 이러한 역할이 더욱 복잡해지면서 자바스크립트의 모듈화와 성능 최적화는 필수적인 과정이 되었습니다.

## 모듈화

모듈화는 코드를 여러 개의 모듈로 분리하는 것을 말합니다. 이를 통해 코드의 재사용성과 유지보수성을 높일 수 있습니다. 자바스크립트는 과거에는 모듈화를 지원하지 않았지만, 최근에는 ES6(ECMAScript 2015)부터 정식으로 모듈 시스템을 도입하였습니다.

ES6의 모듈 시스템은 `import`와 `export` 키워드를 사용하여 모듈 간의 의존성을 관리할 수 있습니다. 모듈화를 적절히 사용하면 코드의 가독성을 높일 수 있으며, 필요한 모듈을 선택적으로 로딩하여 초기 로딩 시간을 단축할 수 있습니다.

예시 코드:

```javascript
// math.js 모듈
export const add = (a, b) => a + b;
export const subtract = (a, b) => a - b;

// main.js 모듈
import { add, subtract } from './math';

console.log(add(3, 5)); // 8
console.log(subtract(10, 4)); // 6
```

## 성능 최적화

자바스크립트의 성능을 최적화하기 위해서는 코드의 실행 시간과 리소스 소모를 줄이는 것이 중요합니다. 다음은 자바스크립트 성능 최적화를 위한 몇 가지 팁입니다.

1. **코드 압축**: 자바스크립트 파일을 압축하여 파일 크기를 줄여 네트워크로부터 전달되는 데이터 양을 줄입니다. 이를 위해 여러 압축 도구와 라이브러리가 제공됩니다.

2. **지연 로딩**: 초기 로딩 시 필요하지 않은 코드를 지연하여 로딩하는 것으로, 웹 페이지의 초기 로딩 속도를 향상시킵니다. 대표적인 방법으로는 비동기 로딩을 사용하는 방법이 있습니다.

3. **DOM 조작 최소화**: DOM 조작은 성능에 큰 영향을 미칩니다. 필요한 경우에만 DOM 조작을 수행하고, 여러 번 반복하는 작업을 최소화하는 것이 좋습니다. 대신, 메모리에 저장된 데이터를 조작하고 한 번에 DOM에 반영하는 방법을 고려할 수 있습니다.

4. **이벤트 처리 최적화**: 이벤트 처리는 페이지가 복잡해질수록 성능 이슈를 야기할 수 있습니다. 이벤트 위임을 사용하거나, 이벤트 리스너를 필요한 범위에만 등록하여 성능을 개선할 수 있습니다.
