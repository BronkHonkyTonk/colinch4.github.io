잘못된 입력에 대한 예외 처리는 소프트웨어 개발에서 중요한 부분 중 하나입니다. 사용자가 부적절한 값을 입력했을 때 시스템이 제대로 대응할 수 있어야 합니다. 이 글에서는 **Shell 스크립트**에서 잘못된 입력에 대한 예외 처리하는 방법에 대해 알아보겠습니다.

## 1. 사용자 입력 검증

사용자로부터 입력을 받을 때, 우선적으로 입력값을 검증해야 합니다. 예를 들어, 숫자를 입력해야 하는데 사용자가 문자열을 입력했을 경우, 이를 검증하여 적절한 에러 메시지와 함께 알려줄 수 있습니다.

아래는 사용자로부터 숫자를 입력받고, 숫자가 아닌 경우 에러 메시지를 출력하는 **Shell 스크립트**의 예제입니다.

```shell
#!/bin/bash
echo "숫자를 입력해주세요: "
read input

if ! [[ $input =~ ^[0-9]+$ ]]; then
    echo "숫자를 입력해주세요."
    exit 1
fi
```

위 예제에서 `^[0-9]+$`는 숫자인지를 검증하는 정규식입니다. 숫자가 아닌 경우 `exit 1`을 통해 스크립트를 종료하고 에러 메시지를 출력합니다.

## 2. 예외 처리

사용자 입력이 검증되고 나면, 이를 통해 예외 처리를 할 수 있습니다. 예를 들어, 특정 조건에서만 실행되어야 하는 스크립트가 있다면, 입력된 값이 조건에 맞지 않을 경우 예외 처리를 할 수 있습니다.

아래는 입력받은 숫자가 5보다 작을 경우 에러 메시지를 출력하는 **Shell 스크립트**의 예제입니다.

```shell
#!/bin/bash
echo "숫자를 입력해주세요: "
read input

if ! [[ $input =~ ^[0-9]+$ ]]; then
    echo "숫자를 입력해주세요."
    exit 1
fi

if [ $input -lt 5 ]; then
    echo "입력한 숫자는 5보다 커야 합니다."
    exit 1
fi
```

위 예제에서는 입력된 값이 5보다 작을 경우 스크립트가 종료되고 에러 메시지가 출력됩니다.

**Shell 스크립트**에서는 이처럼 간단하게 사용자 입력에 대한 예외 처리를 할 수 있습니다. 사용자로부터 입력을 받는 경우, 올바른 값이 입력됐는지를 항상 검증하고, 예외적인 상황에 대해 적절한 대응을 하도록 하는 것이 중요합니다.

이상으로 **Shell 스크립트**에서 잘못된 입력에 대한 예외 처리에 대해 알아보았습니다.
