1. **설정 파일 만들기**: 프로젝트 루트 디렉토리에 `.prettierrc` 파일을 만들고, 코드 스타일에 관한 선호도를 설정합니다.

2. **에디터 플러그인 설치**: 사용 중인 에디터에 Prettier 플러그인을 설치하여 코드 작성 중 자동으로 포맷팅할 수 있습니다.

3. **코드 작성 시 자동 포맷팅**: 저장할 때마다 Prettier가 코드를 자동으로 정리하도록 설정합니다.

4. **충돌 방지**: 동료 개발자들도 Prettier를 사용하도록 유도하여 일관된 코드 스타일을 유지합니다.

Prettier를 사용하면 코드 작성 시 가독성을 높일 수 있으며, 팀 전체의 코드 스타일을 일관되게 유지할 수 있습니다.

[참고 문서](https://prettier.io/docs/en/configuration.html)

정리하자면, Prettier를 사용하여 코드를 작성할 때 가독성을 높이려면 설정 파일을 만들고, 에디터 플러그인을 설치하여 자동 포맷팅 기능을 활용하며 팀원들과의 협업을 통해 일관된 코드 스타일을 유지하는 것이 중요합니다.
