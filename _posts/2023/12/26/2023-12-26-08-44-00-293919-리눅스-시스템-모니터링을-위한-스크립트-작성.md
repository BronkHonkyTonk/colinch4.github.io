리눅스 시스템을 효율적으로 관리하고 문제를 조기에 식별하기 위해서는 시스템 상태와 성능을 지속적으로 모니터링해야 합니다. 이를 위해 스크립트를 작성하여 자동화된 모니터링을 수행할 수 있습니다. 이번 포스트에서는 리눅스 시스템을 모니터링하기 위한 간단한 스크립트를 작성하는 방법에 대해 알아보겠습니다.

## 목표
리눅스 시스템의 CPU, 메모리, 디스크 사용량 및 네트워크 상태를 모니터링하기 위한 스크립트를 작성합니다.

## 스크립트 작성
먼저, **Bash 스크립트**를 사용하여 시스템 모니터링을 위한 스크립트를 작성해보겠습니다.

다음은 각 섹션별로 시스템 상태를 확인하는 스크립트의 일부분입니다.
```bash
# CPU 사용량 확인
cpu_usage=$(top -bn1 | grep "Cpu(s)" | awk '{print $2 + $4}')
echo "CPU 사용량: $cpu_usage%"

# 메모리 사용량 확인
mem_usage=$(free | awk 'NR==2{print $3/($3+$4)*100}')
echo "메모리 사용량: $mem_usage%"

# 디스크 사용량 확인
disk_usage=$(df -h / | awk 'NR==2{print $5}')
echo "디스크 사용량: $disk_usage"

# 네트워크 상태 확인
network_status=$(ping -c 1 google.com &> /dev/null && echo "네트워크 연결: 정상" || echo "네트워크 연결: 이상")
echo $network_status
```

위 스크립트는 각각 CPU, 메모리, 디스크, 네트워크 상태를 확인하는 명령어들을 사용하여 시스템 상태를 모니터링하는 것을 보여줍니다.

## 스크립트 실행
이제 작성한 스크립트를 실행해 보겠습니다. 스크립트를 생성한 후에 `chmod +x monitor.sh` 명령어를 사용하여 실행 권한을 부여한 후, `./monitor.sh`를 실행하여 스크립트를 실행할 수 있습니다.

## 결론
이렇게 하면 간단한 Bash 스크립트를 사용하여 리눅스 시스템을 모니터링하는 데 필요한 기본적인 정보를 확인할 수 있습니다. 이를 통해 시스템의 이상 징후를 빠르게 확인하고 조치를 취할 수 있으며, 시스템 관리의 효율성을 높일 수 있습니다.

이 포스트에서는 간단한 스크립트를 사용한 시스템 모니터링에 대해 다뤘지만, 보다 심화된 모니터링을 위해서는 여러 도구와 기술을 활용할 수 있습니다.
