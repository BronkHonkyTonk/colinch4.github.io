## 소스코드 충돌
팀 프로젝트에서는 여러 명이 동시에 **소스코드를 수정**하고 **병합**하는 경우에 충돌이 발생할 수 있습니다. 충돌은 **동시에 수정된 부분이 겹치는 경우**에 발생하며, 이를 해결하지 않으면 프로젝트를 진행하는 데 큰 불편을 초래할 수 있습니다.

## 해결 방법
1. **최신 변경 내용 반영**: 충돌이 발생하기 쉬운 코드를 수정하기 전에 **병합 대상 브랜치**를 **가져와서 로컬**에서 **최신 변경 내용을 반영**합니다.

    ```c
    git fetch origin
    git checkout [내 브랜치]
    git merge origin/[병합 대상 브랜치]
    ```

2. **충돌 해결**: **병합** 시 충돌이 발생하면 충돌이 발생한 파일을 열어 **수정**하고, 충돌 표시문을 제거한 후에 **변경 내용을 저장**합니다.

3. **병합 완료**: 충돌이 해결되면 **변경 내용을 commit**하고 **병합을 완료**합니다.

    ```c
    git add [수정된 파일]
    git commit -m "병합 완료"
    ```
