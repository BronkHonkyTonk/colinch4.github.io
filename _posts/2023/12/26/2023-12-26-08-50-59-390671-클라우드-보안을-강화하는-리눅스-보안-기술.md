## 배경 정보와 보안 요구 사항

클라우드 환경에서는 데이터의 개인 정보 보호와 시스템 보안이 매우 중요합니다. 가상화 환경과 다양한 사용자가 접근할 수 있는 경로로부터 시스템을 보호하는 것이 필수적입니다.

## SELinux를 통한 보안 강화

**SELinux**는 **리눅스 보안 존재**로, 액세스 제어 보안 정책을 통해 시스템의 보안을 강화합니다. 이를 통해 특권의 권한 부여와 실행 시 권한을 게이트하는 데 유용합니다.

```
sudo setenforce enforcing
```

## AppArmor을 활용한 응용프로그램 보안 강화

**AppArmor**는 **리눅스 커널 보안 모듈**로, 각 프로그램에 대한 보안 프로필을 제공하여 응용프로그램의 행동을 제한할 수 있습니다. 이러한 보안 프로필은 응용프로그램이 사용할 수 있는 시스템 자원과 기능을 특정할 수 있게 해줍니다.

```
sudo aa-complain /usr/bin/firefox
```

## 컨테이너 보안을 위한 Seccomp 설정

**Seccomp**는 컨테이너 내부의 프로세스가 사용할 수 있는 시스템 호출을 제한하여 컨테이너 보안을 강화합니다. 이를 통해 민감한 시스템 호출을 차단하여 보안을 강화할 수 있습니다.

```bash
#include <linux/seccomp.h>
...
scmp_filter_ctx ctx;
ctx = seccomp_init(SCMP_ACT_ALLOW);
seccomp_load(ctx);
```

## 리눅스 보안 모듈 (LSM)

**리눅스 보안 모듈 (LSM)**은 리눅스 커널에 포함된 시스템 보안 플러그인 프레임워크입니다. 이를 사용하여 안전한 클라우드 환경을 조성할 수 있으며, 다양한 보안 모듈을 연동하여 필요한 보안 수준을 확보할 수 있습니다.

이러한 리눅스 보안 기술을 통해 클라우드 보안을 강화할 수 있으며, 사용 시스템의 보안을 위협으로부터 보호할 수 있습니다.
