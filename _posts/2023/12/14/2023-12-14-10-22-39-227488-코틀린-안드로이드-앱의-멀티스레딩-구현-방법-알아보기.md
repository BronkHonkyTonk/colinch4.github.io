## 안드로이드에서의 멀티스레딩이 중요한 이유

안드로이드 앱은 일반적으로 UI 스레드에서 동작하며, **네트워크 요청, 데이터베이스 액세스, 복잡한 계산** 등을 처리해야 합니다. 이러한 작업은 UI 스레드에서 처리되면 앱의 응답성을 떨어뜨리고, 앱이 **무응답** 상태가 될 수 있습니다. 이를 방지하기 위해 백그라운드 스레드에서 작업을 처리하여 앱의 성능을 향상시킬 수 있습니다.

## 코틀린으로의 멀티스레딩 구현

### 스레드를 직접 다루는 방법

코틀린에서는 스레드를 직접 다루는 방법으로 **`Thread`** 클래스를 활용할 수 있습니다. 아래는 스레드를 생성하고 실행하는 간단한 예제 코드입니다.

```kotlin
val myThread = Thread {
    // 스레드에서 수행할 작업 구현
}

myThread.start()
```

### 안드로이드에서 제공하는 멀티스레딩 라이브러리 활용 방법

안드로이드에서는 **`AsyncTask`** 나 **`Handler`** 와 **`Looper`** 등의 멀티스레딩을 지원하는 라이브러리를 제공합니다. 이를 활용하면 보다 편리하게 멀티스레딩을 구현할 수 있습니다.

```kotlin
// AsyncTask를 활용한 백그라운드 작업 예제
class MyAsyncTask : AsyncTask<Unit, Unit, Unit>() {
    override fun doInBackground(vararg params: Unit?) {
        // 백그라운드 작업 수행
    }
}
```
