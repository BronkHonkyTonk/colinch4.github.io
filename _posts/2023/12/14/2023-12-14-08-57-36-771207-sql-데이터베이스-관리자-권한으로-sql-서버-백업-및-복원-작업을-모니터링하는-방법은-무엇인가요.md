SQL Server에서 백업 및 복원 작업을 모니터링하는 것은 매우 중요합니다. 해킹, 데이터 손실 또는 데이터베이스 손상과 같은 문제를 방지하고 신속히 대응하기 위해 이러한 활동을 지속적으로 추적해야 합니다.

### 1. SQL Server 로그 파일 검사

SQL Server Management Studio(SSMS)를 사용하여 로그 파일을 확인하십시오. `LOG` 폴더의 로그 파일을 확인하여 백업 및 복원 작업 기록을 파악할 수 있습니다.

### 2. 시스템 테이블 모니터링

`msdb.dbo.backupset` 및 `msdb.dbo.restorehistory`와 같은 시스템 테이블을 쿼리하여 백업 및 복원 작업에 대한 히스토리를 확인할 수 있습니다.

예시:

```sql
SELECT * FROM msdb.dbo.backupset;
SELECT * FROM msdb.dbo.restorehistory;
``` 

### 3. 이벤트 로그 확인

SQL Server의 이벤트 로그를 검사하여 백업 및 복원 작업과 관련된 이벤트를 체크합니다. 중요한 경고 및 오류 메시지를 식별하여 적절한 조치를 취할 수 있습니다.

### 4. 서드파티 도구 활용

서드파티 SQL Server 관리 도구나 모니터링 도구를 활용하여 백업 및 복원 작업을 손쉽게 추적하고 모니터링할 수 있습니다.

지속적인 백업 및 복원 활동 모니터링은 데이터베이스 보안과 안정성을 강화할 수 있는 중요한 조치입니다. 해당 활동 모니터링을 통해 문제를 조기에 감지하고 조치를 취하여 데이터 손실을 최소화하는 데 도움이 됩니다.
