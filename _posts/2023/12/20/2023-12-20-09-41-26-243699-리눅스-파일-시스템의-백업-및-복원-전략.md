## 리눅스 파일 시스템의 중요성
리눅스 파일 시스템은 컴퓨터 시스템 내의 파일 및 디렉토리를 구성하고 관리하는 체계입니다. 중요한 데이터는 파일 시스템에 저장되며, 장애 발생시 이러한 데이터를 손상 없이 복구할 수 있어야 합니다.

## 백업 전략
### 일괄적 백업
중요한 데이터의 주기적인 **백업**은 시스템 장애 시 데이터를 복구하는데 필수적입니다. 리눅스 시스템에서는 `tar` 명령어를 사용하여 파일 백업을 수행할 수 있습니다.

예시:
```bash
tar -czvf /백업_디렉토리/백업_파일명.tar.gz /백업할_디렉토리
```

### 증분적 백업
시간과 자원을 절약하기 위해 **증분적 백업**을 사용할 수 있습니다. 리눅스에서는 `rsync`를 사용하여 새로 추가되거나 변경된 파일만을 백업할 수 있습니다.

예시:
```bash
rsync -av /원본_디렉토리/ /백업_디렉토리/
```

## 복원 전략
### 전체 복원
시스템 장애 시에는 전체 시스템을 복원해야 할 수도 있습니다. 리눅스 시스템에서는 `tar` 명령어를 사용하여 백업 파일에서 복원을 수행할 수 있습니다.

예시:
```bash
tar -xzvf /백업_파일명.tar.gz -C /복원할_디렉토리
```

### 파일 단위 복원
특정 파일이나 디렉토리만을 복원해야 하는 경우, `rsync` 명령어를 사용하여 원하는 파일이나 디렉토리를 복원할 수 있습니다.

예시:
```bash
rsync -av /백업_디렉토리/원하는_파일_또는_디렉토리/ /원본_디렉토리/
```
