## Ext4 파일 시스템 복구 도구

Ext4 파일 시스템을 사용하는 경우, **e2fsck** 명령어를 사용하여 파일 시스템을 복구할 수 있습니다. 아래는 사용 예시입니다.

```bash
sudo e2fsck -f /dev/sdX
```

위 명령어에서 **/dev/sdX**는 손상된 파일 시스템이 있는 디스크를 나타냅니다. **-f** 옵션은 강제로 복구를 시도하도록 합니다.

---

## XFS 파일 시스템 복구 도구

XFS 파일 시스템을 사용하는 경우, **xfs_repair** 명령어를 사용하여 파일 시스템을 복구할 수 있습니다. 아래는 사용 예시입니다.

```bash
sudo xfs_repair /dev/sdX
```

위 명령어에서 **/dev/sdX**는 손상된 파일 시스템이 있는 디스크를 나타냅니다.

---

## 시스템 복구 시 사용 예시

위에서 소개한 파일 시스템 복구 도구들은 시스템 복구 시 자주 사용됩니다. 아래는 상황에 따른 사용 예시입니다.

- 부팅 중 파일 시스템 오류가 발생한 경우: **e2fsck** 또는 **xfs_repair**을 이용하여 파일 시스템 복구
- 시스템이 정상적으로 부팅되지 않는 경우: 복구 모드에서 **e2fsck** 또는 **xfs_repair**을 이용하여 파일 시스템 복구

이러한 방법들을 통해 파일 시스템에 손상이 발생했을 때, 시스템을 복구할 수 있습니다.
