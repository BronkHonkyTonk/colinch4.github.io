1. **의도를 드러내라**: 메소드 이름은 해당 메소드의 의도를 명확하게 드러내어야 합니다. 다른 프로그래머가 이름만 보고도 해당 메소드가 하는 일을 이해할 수 있어야 합니다.

2. **동사 형태로**: 메소드 이름은 주로 동사 형태로 짓습니다. 메소드가 수행하는 동작을 나타내는 동사로 시작하는 것이 좋습니다.

3. **의미 있는 이름 사용**: 일반적이고 명확한 이름을 사용하여 메소드의 역할을 잘 설명할 수 있도록 합니다.

```csharp
public void CalculateTotalPrice()
{
    // 메소드 내용
}
```

4. **카멜 케이스**: C#에서는 메소드 이름을 카멜 케이스로 작성합니다. 즉, 첫 단어는 소문자로 시작하고 그 뒤에 오는 단어의 첫 글자는 대문자로 시작합니다.

5. **기능에 따라 명확한 이름 부여**: 메소드가 하는 일에 따라 명확한 이름을 부여합니다. 예를 들어 "Get", "Calculate", "Set" 등으로 시작하여 해당 메소드의 기능을 명확히 드러내는 것이 좋습니다.

여기서는 메소드 이름 짓는 규칙에 대해 간략하게 살펴보았습니다. 메소드의 역할과 의도를 잘 표현하여 가독성이 좋고 유지보수하기 쉬운 코드를 작성하는 데 도움이 됩니다.
