리눅스 시스템에서 문제가 발생했을 때 시스템을 복구하고 문제를 진단하기 위해서는 시스템의 상태를 모니터링하는 것이 중요합니다. 이 글에서는 리눅스에서 시스템 복구 중 시스템을 모니터링하는 방법을 알아보겠습니다.

### 1. 시스템 리소스 모니터링

시스템 복구 중에는 **CPU, 메모리, 디스크 사용량** 등의 리소스 사용량을 모니터링 해야 합니다. 이를 위해 리눅스에서는 `top`, `htop`, `vmstat` 등의 명령어를 사용할 수 있습니다.

```bash
$ top
$ htop
$ vmstat
```

### 2. 로그 파일 확인

시스템 복구 중에는 시스템 로그를 확인하여 문제의 원인을 파악해야 합니다. 리눅스에서는 `/var/log` 디렉토리 밑에 다양한 로그 파일들이 저장되어 있습니다. 특히 `syslog`, `kern.log`, `messages` 등의 로그 파일을 주로 확인하게 됩니다.

```bash
$ tail -f /var/log/syslog
$ tail -f /var/log/kern.log
$ tail -f /var/log/messages
```

### 3. 네트워크 및 포트 상태 확인

시스템 복구 중에는 네트워크 연결 상태 및 포트 상태를 확인해야 합니다. 리눅스에서는 `netstat`, `ss` 등의 명령어를 사용하여 네트워크 및 포트 상태를 확인할 수 있습니다.

```bash
$ netstat -tuln
$ ss -tuln
```

### 4. 프로세스 상태 확인

시스템 복구 중에는 현재 실행 중인 프로세스들의 상태를 확인해야 합니다. 리눅스에서는 `ps`, `pstree`, `pgrep` 등의 명령어를 사용하여 프로세스 상태를 확인할 수 있습니다.

```bash
$ ps aux
$ pstree
$ pgrep <process_name>
```

### 5. 시스템 성능 모니터링 도구 활용

시스템 복구 중에는 리눅스 시스템에 기본으로 설치된 성능 모니터링 도구들을 활용할 수 있습니다. 예를 들어 `sar`, `iotop`, `iftop` 등의 도구들을 활용하여 시스템의 성능을 실시간으로 모니터링할 수 있습니다.

리눅스 시스템 복구 중에는 위와 같은 모니터링 방법을 통해 시스템의 상태를 실시간으로 파악하여 문제를 빠르게 해결할 수 있습니다.
