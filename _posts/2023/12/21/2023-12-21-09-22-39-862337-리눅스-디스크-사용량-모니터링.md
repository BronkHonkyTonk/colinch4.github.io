리눅스 시스템에서는 디스크 사용량을 모니터링하는 것이 중요합니다. 디스크 사용량이 너무 높을 경우 시스템 성능이 저하될 수 있기 때문입니다. 

이 블로그에서는 리눅스 시스템에서 디스크 사용량을 모니터링하는 간단한 방법에 대해 살펴보겠습니다.

## 1. df 명령어를 사용한 디스크 사용량 확인
[df](https://www.man7.org/linux/man-pages/man1/df.1.html) 명령어는 파일 시스템의 디스크 사용량을 확인하는 데 사용됩니다. 다음은 df 명령어를 사용하여 현재 디스크 사용량을 확인하는 방법입니다.

```bash
df -h
```

위 명령어를 실행하면 파일 시스템의 디스크 사용량을 휴먼 리더블 형식으로 출력하여 확인할 수 있습니다.

## 2. du 명령어를 사용한 디렉토리별 디스크 사용량 확인
[du](https://www.man7.org/linux/man-pages/man1/du.1.html) 명령어는 디렉토리별 디스크 사용량을 확인하는 데 사용됩니다. 다음은 du 명령어를 사용하여 특정 디렉토리의 디스크 사용량을 확인하는 방법입니다.

```bash
du -h /path/to/directory
```

위 명령어를 실행하면 지정된 디렉토리의 디스크 사용량을 휴먼 리더블 형식으로 출력하여 확인할 수 있습니다.

## 3. cron을 사용한 주기적인 디스크 사용량 모니터링
[cron](https://man7.org/linux/man-pages/man5/crontab.5.html)을 사용하여 주기적으로 df 명령어를 실행하고 결과를 로그 파일에 저장할 수 있습니다. 이를 통해 디스크 사용량의 변화를 추적하고 경고 기준을 설정할 수 있습니다.

```bash
*/5 * * * * /bin/df -h >> /var/log/disk_usage.log
```

위 cron 작업은 매 5분마다 df 명령어를 실행하고 결과를 disk_usage.log 파일에 추가합니다.

리눅스 시스템에서 디스크 사용량을 모니터링하는 방법에 대해 간단하게 살펴보았습니다. 디스크 사용량 모니터링은 시스템 관리자에게 중요한 작업이므로 주기적으로 확인하고 적시에 대응하는 것이 좋습니다.
