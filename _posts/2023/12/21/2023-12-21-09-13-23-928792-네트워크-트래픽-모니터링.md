## 1. 소개

리눅스 시스템에서 네트워크 트래픽을 모니터링하는 것은 시스템 관리자나 네트워크 관리자에게 중요한 과제입니다. 효율적인 네트워크 트래픽 모니터링은 **네트워크 성능을 최적화**하고 잠재적인 문제를 식별하는 데 도움이 됩니다.

## 2. 네트워크 트래픽 모니터링 도구

리눅스 환경에서 네트워크 트래픽을 모니터링하기 위해 다양한 도구가 있습니다. 
- **iftop**: 실시간으로 네트워크 트래픽을 모니터링할 수 있는 인터페이스
- **nload**: 사용 가능한 대역폭을 그래픽으로 표시하는 간단한 모니터링 도구
- **tcpdump**: 네트워크 트래픽을 캡처하고 분석하는 유연하고 강력한 도구

## 3. 네트워크 트래픽 모니터링 방법

### iftop을 사용한 실시간 모니터링

```bash
sudo iftop
```

실시간으로 네트워크 트래픽을 모니터링하려면 터미널에서 `iftop`을 실행합니다. 이를 통해 어떤 호스트가 많은 트래픽을 생성하고 있는지, 특정 포트에 어떤 종류의 트래픽이 발생하는지 확인할 수 있습니다.

### nload를 사용한 대역폭 모니터링

```bash
nload
```

`nload`를 실행하면 사용 가능한 대역폭을 표시하는 간단한 그래픽 인터페이스가 표시됩니다.

### tcpdump를 사용한 트래픽 캡처

```bash
sudo tcpdump -i <interface> -w <output-file>
```

`tcpdump`를 사용하여 네트워크 트래픽을 실시간으로 캡처하고 분석할 수 있습니다. 캡처한 결과는 후속 분석을 위해 파일로 저장할 수 있습니다.

## 4. 결론

리눅스에서 네트워크 트래픽을 모니터링하는 것은 시스템 관리자에게 매우 중요합니다. 다양한 도구를 사용하여 네트워크 성능을 모니터링하고 문제를 식별하여 원활한 네트워크 운영을 도와줍니다.
