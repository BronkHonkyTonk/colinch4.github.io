1. **CPU 사용량**: 테스트 시 CPU 사용량을 모니터링하여 과부하가 발생하지 않도록 합니다. 특히 다중 사용자 환경에서는 CPU 리소스를 효율적으로 사용하는 것이 중요합니다.

2. **메모리 사용량**: 대규모 데이터베이스나 복잡한 쿼리를 실행하는 경우 메모리 사용량을 주의 깊게 모니터링해야 합니다. 메모리 부족으로 인한 성능 하락을 방지하기 위해 적절한 메모리 구성이 필요합니다.

3. **디스크 I/O**: 테스트 중에는 디스크 I/O의 부하를 주의해야 합니다. 데이터베이스가 대규모 트랜잭션을 처리하는 경우 디스크 성능이 병목이 될 수 있으므로 이를 최적화해야 합니다.

4. **네트워크 대역폭**: 분산 환경에서는 네트워크 대역폭을 고려해야 합니다. 데이터 전송 속도가 느릴 경우 테스트 결과가 왜곡될 수 있으므로 네트워크 성능을 최적화해야 합니다.

이러한 서버 리소스 제약 사항을 고려하여 SQL 데이터베이스 성능 테스트를 신중하게 수행해야 합니다.
