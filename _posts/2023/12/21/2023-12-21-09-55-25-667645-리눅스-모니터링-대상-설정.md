리눅스 시스템에서 효과적인 모니터링을 위해서는 적절한 대상을 설정해야 합니다. 리눅스 시스템을 모니터링하는 데에는 여러 가지 방법이 있지만, 여기에서는 주요 리소스와 서비스를 대상으로 모니터링하는 방법에 대해 설명하겠습니다.

## 목차
1. CPU 모니터링
2. 메모리 모니터링
3. 네트워크 모니터링
4. 디스크 모니터링
5. 서비스 상태 모니터링

---

### 1. CPU 모니터링

`top` 명령어를 사용하여 CPU 사용량 및 프로세스별 CPU 사용량을 모니터링할 수 있습니다.

```bash
top
```

### 2. 메모리 모니터링

`free` 명령어를 사용하여 사용 가능한 메모리 용량 및 사용 중인 메모리 용량을 모니터링할 수 있습니다.

```bash
free -m
```

### 3. 네트워크 모니터링

`iftop` 명령어를 사용하여 네트워크 트래픽을 실시간으로 모니터링할 수 있습니다.

```bash
iftop
```

### 4. 디스크 모니터링

`df` 명령어를 사용하여 파일 시스템의 디스크 사용량을 모니터링할 수 있습니다.

```bash
df -h
```

### 5. 서비스 상태 모니터링

`systemctl` 명령어를 사용하여 시스템 서비스의 상태를 모니터링할 수 있습니다.

```bash
systemctl status <service_name>
```

위의 방법들을 통해 리눅스 시스템의 주요 자원 및 서비스를 효과적으로 모니터링할 수 있습니다. 이를 통해 시스템의 안정성과 성능을 지속적으로 관리할 수 있습니다.
