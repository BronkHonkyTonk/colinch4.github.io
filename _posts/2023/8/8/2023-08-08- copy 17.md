Git 충돌이 발생했을 때, 충돌을 해결하지 않고 현재의 상태를 유지하는 방법은 다음과 같습니다:

**주의:** 충돌을 해결하지 않고 현재 상태를 유지하는 것은 일반적으로 권장되는 방법이 아닙니다. 충돌을 해결하지 않고 계속 작업하면 코드의 일관성과 품질을 저하시킬 수 있습니다. 충돌이 발생한 이유를 파악하고 적절한 해결 방법을 선택하는 것이 좋습니다.

1.  **작업 디렉토리를 백업**: 충돌을 해결하지 않고 현재 상태를 유지하려면, 작업 디렉토리의 변경 내용을 보존해야 합니다. 따라서 작업 디렉토리를 다른 폴더에 백업하거나, 변경 내용을 스냅샷 형태로 저장하는 것이 좋습니다.
    
2.  **현재 브랜치에서 커밋하지 않기**: 충돌이 발생한 브랜치에서는 커밋을 하지 않도록 해야 합니다. 이렇게 하면 충돌이 발생하기 전의 상태를 유지할 수 있습니다.
    
3.  **충돌이 발생한 브랜치 이동**: 충돌이 발생한 브랜치를 다른 브랜치로 이동시킵니다. 예를 들어, 현재 브랜치가 'master'이고 충돌이 발생했다면, 다른 브랜치인 'backup' 등으로 이동시킬 수 있습니다.
    
```bash
    git checkout -b backup
```
    
4.  **충돌이 발생하기 전의 커밋으로 이동**: 충돌이 발생하기 전의 커밋으로 이동하여 충돌이 발생하기 전의 상태를 복원합니다.

```bash    
git reset --hard <commit-hash>
```
    
여기서 `<commit-hash>`는 충돌이 발생하기 전의 커밋의 해시값을 의미합니다.
    

위의 방법으로 충돌이 발생하기 전의 상태를 유지할 수 있습니다. 그러나 이후에는 충돌을 해결하거나 새로운 브랜치를 만들어서 작업하는 것이 좋습니다. 충돌을 무시하고 계속 진행하면 코드 품질과 버그 등의 이슈가 발생할 수 있습니다.
