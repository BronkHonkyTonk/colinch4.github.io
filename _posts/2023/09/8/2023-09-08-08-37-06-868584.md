Python에서 디버깅을 할 때 **pdb**(Python Debugger)는 매우 유용한 도구입니다. pdb는 코드 실행 중에 정지점을 설정하고, 코드를 단계별로 실행하며 변수 값 등의 상태를 확인하는 데에 사용됩니다. 

## 모듈 로딩

pdb를 사용하기 위해서는 먼저 사용하고자 하는 모듈을 로딩해야 합니다. 모듈을 로딩하는 방법은 여러 가지가 있지만, 일반적으로 `import` 문을 사용합니다.

```python
import pdb
```

위의 코드는 pdb 모듈을 현재 스크립트에 로딩하는 예시입니다. 이후부터는 pdb 모듈을 사용할 수 있습니다.

## 모듈 관리

pdb를 사용하여 스크립트를 디버깅할 때, 몇 가지 유용한 명령어를 사용할 수 있습니다. 아래는 자주 사용되는 명령어 몇 가지입니다.

- **break** (b): 중단점을 설정합니다. 예를 들어, `break 12`라는 명령어는 12번 줄에서 코드 실행을 중단하도록 설정합니다.
- **continue** (c): 다음 중단점까지 실행을 계속합니다.
- **next** (n): 다음 줄로 넘어갑니다.
- **step** (s): 현재 줄에서 실행되는 함수로 들어갑니다.
- **return** (r): 현재 함수의 실행을 중단하고 호출됐던 위치로 돌아갑니다.
- **list** (l): 현재 위치 주변 코드를 보여줍니다.
- **print** (p): 변수 값을 출력합니다.
- **help** (h): 도움말을 표시합니다.

위의 명령어들은 pdb에서 많이 사용되는 명령어 중 일부입니다. 이 외에도 다양한 명령어를 사용할 수 있으며, 필요에 따라 찾아보시면 됩니다.

## 예제 코드

아래는 간단한 예제 코드입니다. 코드는 두 개의 숫자를 더하는 함수인 `add_numbers`를 정의하고, pdb를 사용하여 디버깅하는 방법을 보여줍니다.

```python
import pdb

def add_numbers(a, b):
    pdb.set_trace()  # 디버깅 모드로 진입
    result = a + b
    return result

num1 = 10
num2 = 20
sum_result = add_numbers(num1, num2)
print("Sum result:", sum_result)
```

위의 코드에서 `pdb.set_trace()`는 디버깅을 시작하는 지점을 설정하는 부분입니다. 코드를 실행하면 `pdb` 프롬프트가 나타나며, 디버깅을 할 수 있는 상태가 됩니다. 

디버깅 모드에서는 `next`, `step`, `print`, `continue` 등의 명령어를 사용하여 코드를 따라가며 디버깅할 수 있습니다. 이를 통해 변수의 값이나 코드의 실행 흐름을 확인할 수 있습니다.

## 결론

pdb는 Python 디버깅에 매우 유용한 도구로, 모듈의 로딩과 디버깅 상태 관리를 효과적으로 수행할 수 있습니다. pdb를 사용하여 코드를 디버깅하면 버그를 찾고 해결하는 데 큰 도움이 됩니다.
