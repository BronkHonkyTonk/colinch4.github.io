이벤트 드리븐 아키텍처(Event-driven architecture)는 비동기 이벤트를 통해 시스템의 다양한 컴포넌트들이 상호작용하는 아키텍처 패턴입니다. 이 패턴은 **이벤트 발생**을 중심으로 시스템을 구축하고, 이벤트를 사용하여 컴포넌트들 간의 통신과 데이터 흐름을 제어합니다.

JSON(JavaScript Object Notation)은 경량 데이터 교환 형식으로, 이벤트 드리븐 아키텍처에서 이벤트 데이터를 표현하는 표준 형식으로 많이 사용됩니다. JSON 형식은 간결하고 가독성이 높아 이벤트 데이터를 쉽게 이해하고 처리할 수 있도록 도와줍니다.

## JSON 기본 구조

JSON은 **키-값 쌍**으로 이루어진 데이터 객체를 표현합니다. 각 키는 문자열로 구성되고, 값은 다양한 데이터 타입을 허용합니다. 다음은 JSON의 기본 구조 예시입니다:

```json
{
  "key1": "value1",
  "key2": "value2",
  "key3": 123,
  "key4": ["item1", "item2", "item3"]
}
```

위 예시에서는 `key1`, `key2`, `key3`, `key4`와 그에 해당하는 값들이 포함되어 있습니다. 값은 문자열, 숫자, 배열 등 다양한 형태일 수 있습니다.

## 이벤트 드리븐 아키텍처와 JSON

이벤트 드리븐 아키텍처에서 JSON은 이벤트를 표현하고 이벤트 데이터를 전달하는 데에 사용됩니다. 이벤트는 시스템 내부 또는 외부에서 발생할 수 있으며, 컴포넌트들 간의 상호작용을 주도합니다. 각 이벤트는 JSON 형식으로 표현되어 해당 이벤트에 대한 정보를 담고 있습니다.

아래는 예시로서 Python에서 JSON을 사용하여 이벤트를 생성하고 전송하는 코드입니다:

```python
import json

event = {
  "eventType": "userLoggedIn",
  "timestamp": "2022-01-01T10:00:00",
  "user": {
    "id": 123,
    "name": "John Doe"
  }
}

event_json = json.dumps(event)

# 이벤트를 전송 또는 특정 컴포넌트로 전달하는 로직
```

위 코드에서는 `event`라는 딕셔너리 객체를 생성하고, JSON 형식으로 변환하여 `event_json` 변수에 저장합니다. 이후에는 해당 이벤트를 원하는 방식으로 전송하거나 다른 컴포넌트로 전달하는 로직을 추가할 수 있습니다.

JSON과 이벤트 드리븐 아키텍처는 데이터의 유연한 표현과 비동기 상호작용을 가능하게 함으로써 다양한 시스템에서 유용하게 사용될 수 있습니다. 이를 활용하여 복잡한 시스템을 설계하고 구축할 때, JSON과 이벤트 드리븐 아키텍처는 매우 유용한 툴이 될 수 있습니다.
