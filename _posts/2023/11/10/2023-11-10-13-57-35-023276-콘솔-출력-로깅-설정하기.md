로깅은 소프트웨어 개발 및 디버깅에서 중요한 역할을 합니다. 콘솔 출력 로깅은 어플리케이션의 실행 중에 발생하는 이벤트, 오류, 경고 등을 개발자가 원하는 형식으로 콘솔에 출력하는 방법입니다. 이를 통해 어플리케이션의 동작 상태를 실시간으로 확인할 수 있습니다.

## 로깅 라이브러리 선택
여러 가지 로깅 라이브러리 중에서는 가장 널리 사용되는 [Log4j](https://logging.apache.org/log4j/2.x/)나 [SLF4J](https://www.slf4j.org/)를 선택할 수 있습니다.

## 콘솔 출력 로깅 설정 방법
다음은 Log4j를 사용하는 경우의 콘솔 출력 로깅 설정 방법입니다.

1. 먼저 Maven이나 Gradle 같은 빌드 도구를 통해 Log4j 의존성을 추가합니다.
```java
// Maven
<dependency>
    <groupId>org.apache.logging.log4j</groupId>
    <artifactId>log4j-core</artifactId>
    <version>2.14.1</version>
</dependency>
```
```groovy
// Gradle
implementation 'org.apache.logging.log4j:log4j-core:2.14.1'
```

2. `log4j2.xml` 또는 `log4j2.properties` 파일을 생성합니다. 이 파일은 로깅 설정을 정의하는데 사용됩니다. 예를 들어, `log4j2.xml` 파일을 아래와 같이 작성합니다.
```xml
<?xml version="1.0" encoding="UTF-8"?>
<Configuration>
    <Appenders>
        <Console name="Console" target="SYSTEM_OUT">
            <PatternLayout>
                <Pattern>%d [%t] %p %c{1.} - %m%n</Pattern>
            </PatternLayout>
        </Console>
    </Appenders>
    <Loggers>
        <Root level="info">
            <AppenderRef ref="Console"/>
        </Root>
    </Loggers>
</Configuration>
```

3. 어플리케이션의 시작 지점에서 로깅 설정을 초기화합니다.
```java
import org.apache.logging.log4j.LogManager;
import org.apache.logging.log4j.Logger;

public class MyApp {
    private static final Logger logger = LogManager.getLogger(MyApp.class);
    
    public static void main(String[] args) {
        // 로깅 설정 초기화
        System.setProperty("log4j.configurationFile", "path/to/log4j2.xml");
        
        // 로그 메시지 출력
        logger.info("Hello, logging!");
    }
}
```

## 마무리
콘솔 출력 로깅 설정은 개발 중에 유용하게 활용할 수 있는 도구입니다. Log4j를 통해 쉽게 로깅 설정을 구성하고 개발자는 어플리케이션의 동작 상태를 실시간으로 확인할 수 있습니다. 라이브러리 선택과 설정 파일 작성 후, `logger`를 통해 로그 메시지를 출력할 수 있습니다. 효과적인 로깅은 디버깅과 모니터링에서 중요한 역할을 하므로 적극 활용하시기 바랍니다.
