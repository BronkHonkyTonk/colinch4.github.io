로그 파일은 시스템 및 애플리케이션 동작 상황을 기록하는 중요한 도구입니다. 하지만 시간이 지나면 로그 파일의 크기가 점점 증가하여 디스크 공간을 차지하게 됩니다. 이로 인해 로그 파일 관리가 필요하며, 로그 파일의 분할 및 압축은 일반적으로 사용되는 방법입니다. 이 글에서는 로그 파일을 효율적으로 분할하고 압축하는 방법을 알아보겠습니다.

## 로그 파일 분할

로그 파일 분할은 큰 로그 파일을 작은 파일로 나누는 작업을 말합니다. 이를 통해 디스크 공간을 효율적으로 관리할 수 있고, 특정 기간에 대한 로그를 검색하거나 분석하기도 편리해집니다. 로그 파일 분할에는 일정 크기 단위로 분할하는 방법과 일정 기간 단위로 분할하는 방법이 있습니다. 예를 들어, 매일 새로운 로그 파일을 생성하거나, 크기가 일정 이상인 로그 파일을 새로운 파일로 분할하는 방법이 있습니다.

> 예시 코드:

public void splitLogFile(String filePath, int maxSizeInBytes) {
    // 로그 파일을 읽어와서 maxSizeInBytes를 기준으로 분할하는 코드 작성
}

## 로그 파일 압축

로그 파일 압축은 로그 파일의 크기를 줄이고 디스크 공간을 절약하기 위한 작업입니다. 압축 알고리즘을 사용하여 로그 파일을 압축하면 파일 크기가 줄어들고 저장 공간을 효율적으로 활용할 수 있습니다. 일반적으로 gzip, zip, 또는 tar와 같은 압축 형식을 사용하여 로그 파일을 압축합니다.

> 예시 코드:

```python
import gzip

def compressLogFile(filePath, compressedFilePath):
    with open(filePath, 'rb') as f_in:
        with gzip.open(compressedFilePath, 'wb') as f_out:
            f_out.writelines(f_in)
```
