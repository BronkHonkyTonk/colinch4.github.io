Druid는 대량의 실시간 데이터를 처리하고 분석하는 데 사용되는 오픈소스 데이터베이스입니다. Druid는 OLAP(온라인 분석 처리) 작업에 특화되어 있으며, 초당 수백만건의 이벤트를 처리할 수 있습니다. 이러한 능력으로 인해 Druid는 대규모 데이터 스트림 처리 및 분석에 많이 사용되고 있습니다.

## 주요 특징
- 실시간 데이터 처리: Druid는 실시간 데이터를 처리하기 위해 설계되었습니다. 데이터가 들어오는 즉시 분석 및 쿼리할 수 있습니다.
- 확장성: Druid는 가로 방향으로 확장 가능한 아키텍처를 가지고 있어 대량의 데이터를 효율적으로 처리할 수 있습니다.
- 쿼리 가능한 스토리지: Druid는 데이터를 색인화하여 쿼리 가능한 형식으로 저장합니다. 이를 통해 빠른 검색 및 분석이 가능합니다.
- 다양한 데이터 유형 지원: Druid는 정형 및 비정형 데이터 모두를 지원하며, 대용량 이벤트 기반 데이터부터 매트릭 데이터까지 다양한 유형의 데이터를 처리할 수 있습니다.

## 사용 사례
- 실시간 분석: Druid는 실시간으로 데이터를 처리하고 분석할 수 있어 실시간 모니터링 및 대시보드 작성에 매우 유용합니다.
- 광고 분석: 대량의 광고 이벤트를 실시간으로 처리하여 광고 성능 분석과 광고 타깃팅에 활용할 수 있습니다.
- IoT 데이터 처리: IoT 장치에서 발생하는 대량의 데이터를 실시간으로 처리하여 실시간 모니터링 및 분석이 가능합니다.

## 예시 코드
```java
import io.druid.query.Query;
import io.druid.query.QueryResult;

// Query 생성
Query query = new Query.Builder()
    .dataSource("my_data_source")
    .granularity("all")
    .intervals("2022-01-01T00:00:00Z/2022-01-02T00:00:00Z")
    .aggregators(Arrays.asList(
        new DoubleSumAggregatorFactory("sum_metric", "metric_field")
    ))
    .build();

// Query 실행
QueryResult result = druidClient.query(query);

// 결과 출력
System.out.println(result);
```
