UnQLite는 C 라이브러리로 제공되는 경량 NoSQL 데이터베이스입니다. 이 데이터베이스는 데이터를 키-값 형태로 저장하며, 내장형 데이터베이스로 사용하기 적합합니다.

## 소개

UnQLite는 Embeddable (내장), Zero-conf (설정 필요 없음), Atomic (원자성 보장), Redis 호환성 제공 등의 특징을 갖고 있습니다. 그리고 이 데이터베이스는 JSON 같은 문서형 데이터와 키-값 데이터를 저장 및 검색할 수 있습니다.

UnQLite는 휴대성이 뛰어나며, 소스 코드가 단일 파일로 되어 있다는 특징이 있습니다. 따라서 다양한 플랫폼에서 쉽게 사용할 수 있습니다. UnQLite는 대규모의 데이터를 처리하는 데에도 높은 성능을 제공하며, 임베디드 시스템이나 모바일 애플리케이션에서도 사용할 수 있습니다.

## 예제

아래는 UnQLite 라이브러리를 사용하여 데이터베이스에 데이터를 저장하고 검색하는 간단한 예제 코드입니다.

```c
#include <unqlite.h>

int main() {
    unqlite *pDb;
    int rc;

    // 데이터베이스 생성
    rc = unqlite_open(&pDb, "mydatabase.db", UNQLITE_OPEN_CREATE);

    if (rc != UNQLITE_OK) {
        fprintf(stderr, "Cannot create database\n");
        return rc;
    }

    // 키-값 저장
    rc = unqlite_kv_store(pDb, "name", -1, "John Doe", -1);

    if (rc != UNQLITE_OK) {
        fprintf(stderr, "Cannot store key-value pair\n");
        return rc;
    }

    // 키-값 검색
    unqlite_int64 nDataLen;
    char *pData;
    rc = unqlite_kv_fetch(pDb, "name", -1, NULL, &nDataLen);

    if (rc == UNQLITE_OK) {
        pData = malloc(nDataLen);
        rc = unqlite_kv_fetch(pDb, "name", -1, pData, &nDataLen);

        if (rc == UNQLITE_OK) {
            printf("Value: %s\n", pData);
        }

        free(pData);
    }

    // 데이터베이스 닫기
    unqlite_close(pDb);

    return 0;
}
```

## 요약

UnQLite는 경량 NoSQL 데이터베이스로서, 휴대성이 좋고 성능이 우수합니다. 데이터를 키-값 형태로 저장하고 검색할 수 있으며 다양한 플랫폼에서 사용할 수 있습니다. 자세한 내용은 [UnQLite 공식 홈페이지](https://unqlite.org/)를 참조하십시오.
