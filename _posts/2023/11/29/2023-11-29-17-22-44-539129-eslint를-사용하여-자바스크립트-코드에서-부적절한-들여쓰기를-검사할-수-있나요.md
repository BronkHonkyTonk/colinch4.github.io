들여쓰기는 코드의 가독성과 유지보수성을 향상시키는 데 중요한 역할을 합니다. 부적절한 들여쓰기는 코드의 가독성을 낮추고 버그 발생 가능성을 높입니다. ESLint를 사용하여 코드에서 일관된 들여쓰기 규칙을 적용하고 이러한 문제를 자동으로 검사할 수 있습니다.

ESLint를 사용하려면 먼저 프로젝트에 ESLint를 설치해야 합니다. 이를 위해서는 npm이나 yarn과 같은 패키지 매니저를 사용하여 ESLint를 설치하면 됩니다. 설치가 완료되면 프로젝트의 루트 디렉토리에 `.eslintrc.json` 또는 `.eslintrc.js`라는 파일을 생성하여 ESLint 구성을 설정합니다.

들여쓰기 검사를 활성화하려면 해당 구성 파일에 들여쓰기 규칙을 추가하면 됩니다. 예를 들어, 다음과 같은 들여쓰기 규칙을 설정할 수 있습니다:

```javascript
{
  "rules": {
    "indent": ["error", 2]
  }
}
```

위의 예제에서 `"indent"`는 들여쓰기 규칙을 나타내며, `"error"`는 부적절한 들여쓰기를 오류로 간주하도록 설정한 것입니다. `"2"`는 들여쓰기에 사용할 공백의 수를 나타냅니다.

이제 코드를 실행하면 ESLint가 설정한 들여쓰기 규칙을 적용하여 코드를 검사합니다. 부적절한 들여쓰기가 있는 경우 ESLint는 오류 메시지를 표시하여 해당 문제를 식별할 수 있도록 도와줍니다.

ESLint를 통해 자바스크립트 코드의 들여쓰기를 검사함으로써 일관된 코드 스타일을 유지하고 오류를 방지할 수 있습니다. 이는 코드의 가독성과 유지보수성을 향상시키는 데 큰 도움이 됩니다.
