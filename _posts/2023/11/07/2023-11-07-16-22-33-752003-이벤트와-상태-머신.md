소프트웨어 개발에서 이벤트와 상태 머신은 매우 중요한 개념입니다. 이벤트는 시스템에 발생하는 액션이나 변화를 나타내는 것이며, 상태 머신은 시스템이 이벤트에 응답하여 상태를 전환하는 방식을 정의합니다.

## 이벤트

이벤트는 시스템에서 발생하는 액션 또는 변화를 나타냅니다. 예를 들어, 사용자의 버튼 클릭, 센서에서의 데이터 수집, 외부 시스템과의 통신 등이 이벤트의 예입니다. 이벤트는 시스템의 동작을 트리거하고 다른 상태로 전환될 수 있도록 설정됩니다.

## 상태 머신

상태 머신은 시스템의 동작을 정의하는 모델입니다. 이 모델은 상태와 이벤트 간의 관계를 다이어그램으로 표현할 수 있으며, 상태 전환 로직을 명확하게 나타냅니다. 상태는 시스템의 동작 상태를 의미하며, 이벤트에 응답하여 상태가 변화할 수 있습니다. 

### 상태 머신 다이어그램

상태 머신 다이어그램은 상태와 이벤트 간의 관계를 그래픽으로 나타낸 다이어그램입니다. 이 다이어그램은 상태와 이벤트의 전이를 명확하게 보여주어 시스템의 동작을 이해하기 쉽게 합니다. 상태 머신 다이어그램은 UML(Unified Modeling Language)의 한 부분으로 널리 사용됩니다.

## 예시 코드

```python
class TrafficLight:
    def __init__(self):
        self.state = 'green'
        
    def change_state(self, event):
        if event == 'button_pressed':
            if self.state == 'green':
                self.state = 'red'
            elif self.state == 'red':
                self.state = 'yellow'
            else:
                self.state = 'green'
                
    def get_state(self):
        return self.state
```

위의 예시 코드는 교통 신호등을 상태 머신으로 모델링한 것입니다. 상태는 'green', 'red', 'yellow'로 나뉘어지며, 버튼 클릭 이벤트에 따라 상태가 변경됩니다.
