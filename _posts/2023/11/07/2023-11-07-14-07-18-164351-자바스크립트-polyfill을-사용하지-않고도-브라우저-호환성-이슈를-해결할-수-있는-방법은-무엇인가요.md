**1. Feature detection (기능 감지):** 해당 기능을 사용하기 전에 브라우저가 해당 기능을 지원하는지를 감지하여 분기 처리할 수 있습니다. 예를 들어, `Array.prototype.map`을 사용하기 전에 다음과 같이 감지할 수 있습니다.

```javascript
if (!Array.prototype.map) {
    Array.prototype.map = function(callback) {
        // 구현 로직
    };
}
```

**2. Progressive enhancement (점진적 향상):** 모든 브라우저에서 동작하는 기본적인 동작은 구현한 후, 추가적인 기능을 지원하는 브라우저에 대해서만 더 향상된 기능을 적용합니다. 예를 들어, 기본적인 HTML 폼은 폼 전송이 가능해야 하며, 더 향상된 기능을 지원하는 브라우저에 대해서만 AJAX를 활용한 비동기적인 폼 전송을 추가로 구현할 수 있습니다.

**3. 라이브러리나 프레임워크 활용:** 대부분의 자바스크립트 라이브러리나 프레임워크는 브라우저 호환성 이슈를 고려한 기능을 제공합니다. 이러한 라이브러리나 프레임워크를 사용하면 별도의 호환성 대응 작업을 하지 않아도 되며, 모든 브라우저에서 동일한 동작을 할 수 있습니다.

이러한 방법들을 적절히 조합하여 브라우저 호환성 이슈를 해결할 수 있습니다. 하지만 모든 경우에 대해 호환성을 보장하기는 어렵기 때문에, 실제 사용자의 브라우저 상황을 고려하여 테스트해보는 것이 중요합니다.
