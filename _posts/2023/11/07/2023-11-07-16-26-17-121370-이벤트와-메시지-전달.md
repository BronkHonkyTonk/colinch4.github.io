## 이벤트란?

이벤트는 시스템에서 발생하는 특정한 사건을 나타냅니다. 예를 들어, 사용자가 버튼을 클릭하면 이는 "클릭" 이벤트로 간주됩니다. 이벤트는 사용자 입력, 센서 파악, 네트워크 통신 등 다양한 상황에서 발생할 수 있습니다.

## 메시지 전달이란?

메시지 전달은 시스템의 다른 부분 또는 컴포넌트 간에 정보를 교환하는 방법입니다. 이는 일방적인 전달 방식으로, 송신자가 메시지를 보내고 수신자가 메시지를 받아 처리하는 형태를 취합니다. 메시지 전달은 비동기적인 방식으로 이뤄지며, 송신자와 수신자 간의 결합도를 낮추어 유연성을 제공합니다.

## 이벤트와 메시지 전달의 사용 이유

이벤트와 메시지 전달의 사용 이유는 다음과 같습니다:

1. 느슨한 결합: 이벤트와 메시지 전달은 시스템 내의 컴포넌트 간의 결합도를 낮춥니다. 이는 컴포넌트를 독립적으로 개발하고 테스트할 수 있게 해줍니다.

2. 확장성: 이벤트와 메시지 전달은 시스템에 새로운 동작을 추가하는 데 용이합니다. 새로운 컴포넌트를 추가하거나 기존 컴포넌트의 동작을 변경하고자 할 때, 이벤트나 메시지를 사용하여 유연하게 확장할 수 있습니다.

3. 비동기 통신: 이벤트와 메시지 전달은 비동기적인 통신 방식을 제공합니다. 이는 여러 컴포넌트 간에 동시에 작업을 수행할 수 있게 하며, 응답 시간을 최적화할 수 있습니다.

## 예시 코드

아래는 JavaScript에서 이벤트 및 메시지 전달을 다루는 간단한 예시 코드입니다:

```javascript
// 이벤트 핸들러 등록
document.getElementById('myButton').addEventListener('click', function() {
    // 버튼 클릭 이벤트 처리
    console.log('버튼이 클릭되었습니다!');
});

// 메시지 전달
function sendMessage(receiver, message) {
    receiver.receiveMessage(message);
}

// 메시지 수신자
const receiver = {
    receiveMessage: function(message) {
        console.log('메시지를 수신했습니다:', message);
    }
};

// 메시지 전달 호출
sendMessage(receiver, '안녕하세요!');
```

이 코드에서는 버튼 클릭 이벤트를 등록하고, 메시지 전달 함수를 정의하여 메시지 수신자에게 메시지를 전달합니다.
