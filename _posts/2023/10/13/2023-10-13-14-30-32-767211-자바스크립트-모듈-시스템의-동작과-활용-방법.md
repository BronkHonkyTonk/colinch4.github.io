자바스크립트 모듈 시스템은 코드를 모듈화하고 재사용 가능한 코드 단위로 나누는 기능을 제공하는 도구입니다. 이를 통해 프로젝트의 구조를 단순화하고 코드의 가독성과 유지 보수성을 향상시킬 수 있습니다. 이번 글에서는 자바스크립트 모듈 시스템의 동작 방식과 그 활용 방법에 대해 알아보겠습니다.

## 1. 자바스크립트 모듈 시스템의 동작 방식

자바스크립트 모듈 시스템은 모듈을 정의하고 내보내는 방식으로 동작합니다. 각 모듈은 독립적인 네임스페이스를 가지며, 필요한 코드만 외부로 공개할 수 있습니다. 다른 모듈은 해당 모듈에서 내보낸 것들을 가져와서 사용할 수 있습니다.

### 1.1. 모듈 정의하기

자바스크립트 모듈은 일반적으로 파일 하나에 하나의 모듈을 정의하는 형태로 사용됩니다. 모듈은 `export` 키워드를 사용하여 외부로 공개할 코드를 선언합니다.

```javascript
// 모듈 정의하기
// myModule.js

export const PI = 3.14;

export function multiply(a, b) {
  return a * b;
}

export default function sayHello() {
  console.log("Hello!");
}
```

### 1.2. 모듈 가져오기

모듈을 사용하기 위해서는 `import` 문을 사용하여 해당 모듈을 가져와야 합니다. 가져온 모듈은 원하는 변수에 할당하여 사용할 수 있습니다.

```javascript
// 모듈 가져오기
// main.js

import { multiply, PI } from './myModule.js';
import sayHello from './myModule.js';

console.log(multiply(2, 3)); // 6
console.log(PI); // 3.14
sayHello(); // Hello!
```

## 2. 자바스크립트 모듈 시스템의 활용 방법

### 2.1. 모듈화된 코드의 장점

자바스크립트 모듈 시스템을 활용하면 코드의 재사용성과 유지 보수성을 높일 수 있습니다. 각각의 모듈은 독립적인 네임스페이스를 가지기 때문에, 변수 또는 함수의 이름 충돌 문제를 피할 수 있습니다. 또한 필요한 모듈만 가져와서 사용할 수 있기 때문에, 전체 코드를 로드할 필요 없이 필요한 부분만 가져와서 실행할 수 있습니다.

### 2.2. 외부 라이브러리 가져오기

자바스크립트 모듈 시스템은 외부 라이브러리의 가져오기에도 사용될 수 있습니다. `import` 문을 사용하여 필요한 라이브러리를 가져올 수 있으며, 해당 라이브러리의 기능을 활용할 수 있습니다.

```javascript
import axios from 'axios';

axios.get('/api/data')
  .then(response => {
    console.log(response.data);
  })
  .catch(error => {
    console.error(error);
  });
```

## 마무리

자바스크립트 모듈 시스템은 코드를 모듈화하고 재사용 가능한 코드 단위로 나누는 기능을 제공하여 프로젝트의 구조를 단순화하고 유지 보수성을 높일 수 있습니다. 모듈을 정의하고 가져오는 방식으로 모듈 시스템을 활용할 수 있으며, 외부 라이브러리를 가져오기 위해서도 해당 방식을 사용할 수 있습니다.
